REST Endpoints

Standard JSON endpoints for markets, bets, suggestions, and oracle events. Authenticated endpoints expect wallet headers with the active chain (`aptos` or `sui`). 

GET    /api/markets                  # list markets
GET    /api/markets/:chain/:id       # market details
POST   /api/markets/calculate-payout # payout estimate (requires wallet headers)
POST   /api/s suggestions            # create market suggestion (authenticated)
POST   /api/auth/nonce               # sign-in nonce for wallet auth

GraphQL & Indexer

Use the Postgres-backed Prisma layer or the event indexer GraphQL gateway for analytics workloads. 

# Example query
query MarketsByChain($chain: String!) {
  markets(where: { chain: { equals: $chain } }, orderBy: { createdAt: desc }) {
    id
    question
    outcomes
    totalVolume
    endDate
  }
}

Enable the optional GraphQL server by setting ENABLE_GRAPHQL=true in backend/.env.

Authentication Headers

Protected endpoints expect the active wallet address and chain in request headers. Sessions expire by default after 5 minutes. 

POST /api/markets/calculate-payout
Headers:
  Content-Type: application/json
  X-Wallet-Address: 0xabc...
  X-Active-Chain: aptos // or "sui"
Body:
{
  "marketId": "93f6...",
  "outcomeIndex": 1,
  "amount": 250
}

SDK Helpers

Prefer the TypeScript SDK when integrating from React/Next.js apps. It handles wallet flows, Move transactions, and REST calls. 

import { MoveMarketSDK } from '@/services/MoveMarketSDK';

const sdk = new MoveMarketSDK('testnet', MODULE_ADDRESS);
const markets = await sdk.listMarkets();
const quote = await sdk.calculatePayout(marketId, outcomeIndex, 100);
